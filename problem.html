Describe your problem or issue, then click a button to see the formatted data in the Result Window. Click "select all" and copy. Finally, paste into an email to be sent to the engineering coordinator. Alternatively, click the Problem Database special link and paste into the product description window.

Three different format buttons are provided:- HTML, Plain, and Wiki. HTML looks neater if it is saved in a file, such as "test.html" and reloaded. Plain text will work for any situation including Bugzilla forms. Wiki is simple and useful but requires demonstration until the concept is better understoud.

One must copy the data from the result window because of Javascipt security. It prevents writing files or sending email directly from the form when no web server is present. One can print this form, but the data will not appear on the printout.

The "Clear" button erases all input data. Optionally use it to clear the form before you enter any information. Once you have entered your data, you need not clear the form since the other buttons cause the result window to be refreshed with each format.

Summary: Top
Start with a one-line summary like "Component does not work". Examples follow:-
  • Laser does not fire properly
  • Alert xxx document is confusing
  • Sensor head cable is broken
  • Optical signal is degraded
  • ALTM time is wrong
Description: Top
Describe the problem or name a document containing report logs and other related material. Typing hints: Use the TAB or shift-TAB to navigate the form. The text window is 60 columns. Press ENTER when typing near the right edge so that the result will appear in block format.
Product Number:  Top
Enter product number, a project number, or perhaps, a customer number. If that is not known, enter "NA".
Serial Number:  Top
Enter a number or token which further identifies the product or project. Perhaps this could be a ticket number from another reporting system. If the number does not matter, enter "NA".
Reporter:  Top
Enter the email alias of the person reporting the problem. For example, "bobmc". Please do not use initials or arbitrary formats.
